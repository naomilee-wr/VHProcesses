Story Process¶


This process describes how stories are created, implemented and completed. The Scrum team is responsible for executing this process. The steps in this process are from the Release Planning Process, Sprint Planning Process, and Daily Sprint Process. The steps are repeated here to make the end to end story process clear.


Entry Criteria

The PM (Product Owner) has market input for a product release.

Inputs

Marketing and Engineering input.

Exit Criteria

Stories have been written, estimated, implemented, and accepted.

Outputs

Stories are in the accepted state in the Requirements Management system (e.g., Jira Agile)

Activities¶

Step #

Activity Name

Description

1

Plan the Release

The Scrum team and Key Stakeholders (e.g., EPM, PM, TFO, Dev & Test Managers, Architects) work together to plan the release. The release strategy is defined and planned using the Release Planning Process.

2

Plan the Sprint

The Scrum team and Key Stakeholders (e.g., EPM, PM, TFO, Dev & Test Managers, Architects) work together to plan the sprint activities. The sprint is planned using the Sprint Planning Process.

The activities at a minimum include:

  • groom the backlog

  • understand and deconstruct the Stories

  • Optionally, write/estimate/assign tasks for Stories. This is optional. (See Checklist/Guideline for Story Preparation)

  • determine what fits in the sprint

3

Implement stories

Team members work on Stories and update their status.

4

Accept stories

When the acceptance criteria is achieved, the Tehcnical Feature Owner (TFO) or designee accepts (approves) Stories that are complete according to the Definition of Done for the Story.

Refer to Checklist/Guideline for Story Preparation.
